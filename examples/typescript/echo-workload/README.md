# echo-workload (TypeScript)

Minimal ACTR EchoService workload authored in TypeScript with protobuf code
generated by `actr gen -l typescript`, compiled to a `wasm32-wasip2`
Component Model module, and packaged as a signed `.actr`.

This example uses the package-first workload authoring package at
`../../../bindings/typescript/actr-workload`; no retired TypeScript workload SDK
is required.

## What It Does

- Exports `acme:EchoService:0.1.0`.
- Handles `echo.EchoService.Echo`.
- Keeps generated dispatch glue in `src/generated/echo_workload.ts`.
- Keeps user business logic in `src/actr_service.ts`, where the handler
  implements `EchoServiceHandler` and delegates dispatch to
  `EchoServiceDispatcher`.

## Build

```bash
npm install
npm run build
npm run componentize
```

The output component is:

```text
dist/echo-typescript-0.1.0-wasm32-wasip2.wasm
```

## Regenerate Protobuf TypeScript

```bash
npm run codegen
```

The checked-in generated sources under `src/generated` are intentional smoke
coverage: they must build/import without depending on stale workload runtime
APIs.

## Packaging

```bash
npm run package -- --key dist/dev-key.json
```

The package step runs `actr build --no-compile`. Compilation is handled by the
TypeScript workload componentizer.
