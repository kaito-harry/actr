# C echo workload

<!-- SPDX-License-Identifier: Apache-2.0 -->

This example implements the async
[`actr-workload-guest-v2`](../../../core/framework/wit-v2/actr-workload.wit)
world (`actr:workload@0.2.0`) in C. `dispatch` returns the inbound payload with
an `echo: ` prefix; lifecycle and observation hooks complete immediately.

Every export receives the V2 `invocation-ctx`. The implementation releases
owned arguments on every path and completes each asynchronous export through
the task-return helper generated by `wit-bindgen`.

## Pinned toolchain

| Tool | Version |
| --- | --- |
| `wit-bindgen` | 0.59.0 |
| `wasi-sdk` | 24 |
| `wasm-tools` | 1.253.0 |
| Wasmtime Preview 1 reactor adapter | 46.0.1 |

Download `wasi_snapshot_preview1.reactor.wasm` from the Wasmtime 46.0.1
release. CI verifies all downloaded archives and the adapter by SHA-256.

The build follows the upstream C component pipeline:

```text
wit-bindgen c
  -> clang (wasm32-wasip1 reactor)
  -> wasm-tools component new (Preview 1 adapter)
  -> wasm-tools validate
```

## Build

```bash
make \
  WASI_SDK=/opt/wasi-sdk-24 \
  WASI_ADAPTER=/path/to/wasi_snapshot_preview1.reactor.wasm
```

Useful individual targets are `bindings`, `component`, `verify`, and `clean`.
The final component is written to:

```text
dist/echo-c-0.1.0-wasm32-wasip2.wasm
```

`make verify` validates the binary and checks that its inferred world exports
`actr:workload/workload@0.2.0`, uses async functions and explicit invocation
context, and contains no V1 interface. Because this echo implementation does
not call a host function, componentization trims the unused `host` import.

## Memory ownership

- Inbound records containing strings or lists are guest-owned. The
  implementation releases each with the generated `*_free` helper.
- The response byte list is allocated with `malloc` and transferred to the
  host by the generated dispatch task-return helper.
- Scalar-only event records do not own allocations and need no free helper.

Generated bindings, intermediate core Wasm, and final artifacts are excluded
from version control and reproduced by CI.
